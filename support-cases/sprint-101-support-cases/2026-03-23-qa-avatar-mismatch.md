# QA Avatar Mismatch in Assigned Stories

## Date
2026-03-23

## Context

At the beginning of Sprint 101, a QA issue was raised where some assigned stories were showing the wrong profile image for the responsible user.

The visible name matched the assigned person, but the avatar belonged to a different user, indicating a synchronization or data-linking issue between the user identity and the asset shown in the interface.

---

## Actions Taken

- Reviewed the issue directly in the QA Dashboard
- Validated the mismatch between the assigned user name and the displayed avatar
- Documented the impact on daily project identification and internal trust in the workflow
- Registered the case as a data/database issue for follow-up

---

## Result

The issue was formally documented in the QA platform with enough detail to support investigation and correction in a later step.

---

## Time Spent

0.5 hours
